What is Shannon Insight?

Code is a signal. Measure it like one.

Named after Claude Shannon, the father of information theory, Shannon Insight treats source code as a measurable physical system. It computes 62 quantitative signals for every file, module, and codebase -- then surfaces actionable findings backed by mathematical evidence.

Traditional tools check syntax rules. Shannon Insight measures the information content, dependency topology, spectral properties, and temporal evolution of your codebase. It finds what linters, complexity analyzers, and coverage tools cannot see.

A file can pass every lint rule yet be architecturally dangerous -- imported by 47 files, modified 200 times, maintained by a single developer. No single-dimension tool flags this. Shannon Insight does.

Mathematical Foundations

Rigorous math, not heuristics

Every finding is backed by quantitative signals with defined formulas, explicit thresholds, and computed confidence scores.

Information Theory

Shannon Entropy

H(X) = -∑ p(x) · log₂(p(x))

Measures the average information content. Applied to authorship distributions: H=0 means one person wrote everything. H=log₂(k) means k authors contributed equally. Bus factor = 2H.

Graph Theory

PageRank

PR(v) = (1-d)/N + d · ∑ PR(u)/deg(u)

Computes importance in the dependency network. Files imported by many important files get high PageRank. Damping factor d=0.85, power iteration convergence at 10-6.

Spectral Theory

Health Laplacian

Δh(v) = h(v) - (1/|N(v)|) · ∑ h(u)

The discrete Laplacian applied to the risk scalar field. Positive Δh means the file is a local weak point -- much worse than its graph neighbors. Unique to Shannon Insight.

Compression

Normalized Compression Distance

NCD(x,y) = (C(xy) - min(C(x),C(y))) / max(C(x),C(y))

Information-theoretic clone detection. NCD approximates the Normalized Information Distance using a real compressor. Pairs with NCD < 0.3 are flagged as copy-paste clones.

Quick Start

Up and running in 60 seconds

  • 1

    Install

    One pip command. No server. No cloud. Everything runs locally.

  • 2

    Analyze

    Point it at any project. 8 languages supported out of the box.

  • 3

    Track

    Save snapshots, compare diffs, and monitor health trends over time.

  • 4

    Automate

    JSON output and exit codes integrate directly into CI/CD pipelines.

# Install
$ pip install shannon-codebase-insight
 
# For enhanced parsing (recommended)
$ pip install shannon-codebase-insight[parsing]
 
# Analyze a project
$ shannon-insight /path/to/project
 
# Save snapshot for trend tracking
$ shannon-insight /path/to/project --save
 
# CI/CD quality gate
$ shannon-insight . --json --fail-on high
Use Cases

Built for real engineering workflows

Shannon Insight integrates into the way your team already works.

Code Review Automation

Run on changed files with --changed to focus review attention on structurally risky changes, not just formatting issues.

shannon-insight . --changed --json

Technical Debt Tracking

Save snapshots over time and track codebase health trends. See which modules are improving, which are eroding, and where to invest effort.

shannon-insight health /path/to/project

Onboarding New Developers

Knowledge silos, critical hubs, and community structure maps give new team members a structural understanding of the codebase in minutes.

shannon-insight explain src/core/engine.py

Architectural Governance

Detect layer violations, zones of pain, and boundary mismatches automatically. Enforce architecture decisions without manual review.

shannon-insight . --fail-on high

CI/CD Quality Gates

Fail builds when high-severity findings appear. JSON output integrates with any CI system. Exit codes signal pass/fail.

shannon-insight . --json --fail-on any

AI Code Quality Audit

Detect patterns common in AI-generated codebases: orphan files, hollow code, phantom imports, flat architecture, copy-paste clones. The wiring score captures this signature.

shannon-insight . --verbose
